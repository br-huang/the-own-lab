# SSH

## SSH Config 結構

### 全局設定 (`Host *`)

```
Host *
  AddKeysToAgent yes    # SSH 連線時自動將鑰匙載入 agent
  UseKeychain yes       # 允許 SSH 讀取 macOS Keychain
  IdentityFile ~/.ssh/id_ed25519  # 預設私鑰
```

### 主機設定範例

```
Host tbd-120
  HostName 192.168.1.120       # 實際 IP
  User tbd                      # 登入用戶
  ForwardAgent yes              # 允許跳板轉發 agent
  AddKeysToAgent yes
  IdentityFile ~/.ssh/Server-120  # 指定密鑰
  IdentitiesOnly yes              # 只用指定的密鑰，不嘗試其他
```

- `IdentitiesOnly yes`：只嘗試 `IdentityFile` 指定的密鑰，避免 agent 中的其他鑰匙被逐一嘗試
- `ForwardAgent yes`：允許遠端主機透過你的本機 agent 簽名（跳板機場景），生產環境不建議開啟

## Passphrase 的目的

私鑰檔案就是身份證明，Passphrase 是**對私鑰檔案本身的加密**。

|        | 沒有 passphrase | 有 passphrase |
| ------ | ------------- | ------------ |
| 私鑰檔案狀態 | 明文存在磁碟上       | AES 加密存在磁碟上  |
| 檔案被偷走  | 對方直接能用，冒充你    | 對方打不開，沒用     |

> 比喻：私鑰是家門鑰匙，passphrase 是裝鑰匙的保險箱密碼。

## ssh-agent

### 是什麼

ssh-agent 是一個 **daemon**（守護進程）— 在背景默默運行的程式，沒有畫面、不需要互動。

它做一件事：**把解密後的私鑰存在記憶體中，需要簽名時代簽**。

### 沒有 agent vs 有 agent

```
沒有 agent：
每次 SSH → 讀取私鑰檔案 → 要求輸入 passphrase 解密 → 連線（重複再重複）

有 agent：
第一次：輸入 passphrase → 解密後的私鑰交給 agent 保管（記憶體中）
之後：SSH → agent 直接代簽 → 連線（不再問你）
```

### 實現原理（虛擬碼）

```python
# ===== ssh-agent 核心 =====

key_store = {}  # { fingerprint: decrypted_private_key }

# 建立 Unix Domain Socket（本機進程間通訊，不走網路）
sock = socket.socket(AF_UNIX)
sock.bind("/tmp/ssh-XXXX/agent.12345")
os.chmod(sock, 0o600)  # 只有本用戶能存取

# 環境變數讓 ssh 知道去哪找 agent
os.environ["SSH_AUTH_SOCK"] = "/tmp/ssh-XXXX/agent.12345"

# 主迴圈
while True:
    client = sock.accept()
    msg = client.recv()

    if msg.type == SSH_AGENTC_ADD_IDENTITY:
        # ssh-add 送來的：「請保管這把鑰匙」
        key_store[msg.key.fingerprint] = msg.private_key
        client.send(SSH_AGENT_SUCCESS)

    elif msg.type == SSH_AGENTC_SIGN_REQUEST:
        # ssh 送來的：「請用某把鑰匙簽名這段資料」
        key = key_store[msg.fingerprint]
        signature = crypto_sign(key, msg.data)
        client.send(signature)  # 只回傳簽名，私鑰永不外洩

    elif msg.type == SSH_AGENTC_REQUEST_IDENTITIES:
        # 「你手上有哪些鑰匙？」（ssh-add -l）
        client.send(list(key_store.public_keys()))
```

### 安全設計

```
ssh 客戶端 ──「幫我用 key X 簽名」──→ ssh-agent
                                         │
                                    key_store (記憶體)
                                    ┌───────────────┐
                                    │ 私鑰 A (解密) │
                                    │ 私鑰 B (解密) │
                                    └───────────────┘
                                         │
ssh 客戶端 ←── 只回傳簽名結果 ────────────┘
               ❌ 絕不回傳私鑰本身
```

## ssh-add 做了什麼

```python
def ssh_add(key_path, passphrase_source="terminal"):
    # 1. 讀取加密的私鑰檔案
    encrypted_key = read_file(key_path)

    # 2. 取得 passphrase
    if passphrase_source == "keychain":
        passphrase = keychain.lookup(key_path)
    else:
        passphrase = prompt_user("Enter passphrase: ")

    # 3. 用 passphrase 解密私鑰
    private_key = decrypt(encrypted_key, passphrase)

    # 4. 透過 Unix socket 送給 agent
    sock = connect(os.environ["SSH_AUTH_SOCK"])
    sock.send(SSH_AGENTC_ADD_IDENTITY, private_key)

    # 5. agent 收到後存入記憶體，ssh-add 丟棄私鑰
    del private_key
```

### 加入 Keychain 版本

```bash
ssh-add --apple-use-keychain ~/.ssh/id_ed25519
```

輸入一次 passphrase 後，macOS Keychain 會記住，重開機後也不用再輸入。

## SSH 連線完整流程

```
ssh young-linux-su
      │
      ▼
1. TCP 連線到 192.168.8.34:22
      │
      ▼
2. 伺服器發來 challenge（隨機資料）
   「證明你擁有這把公鑰對應的私鑰」
      │
      ▼
3. ssh 客戶端檢查：
   ├─ 有私鑰（無 passphrase）→ 直接簽名
   ├─ 有私鑰（有 passphrase）→ 問你輸入 / 問 keychain
   └─ 沒有 → 問 ssh-agent 代簽
      │
      ▼
4. 簽名送回伺服器 → 伺服器用公鑰驗證 → 通過 → 登入成功
```
